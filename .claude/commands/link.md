# /link

Run a semantic linking pass over the wiki. The goal is to strengthen the
knowledge graph with **verified** `[[wikilinks]]` and propose Maps of
Content (MOCs) for thematic clusters — without hallucinating links to
articles that do not exist.

This command **never modifies existing wiki articles directly**. It writes
a draft to `wiki/link-pass-{YYYY-MM-DD}.md`. Only after the founder
approves do you apply the changes.

## When to run

Founder invokes this when:

- The wiki has grown but cross-links are sparse
- New articles were added without bidirectional linking
- A thematic cluster needs a MOC for navigation
- During `/reflect` if you notice orphan or weakly connected articles

If the wiki has fewer than 2 articles, decline gracefully — not enough
to link yet.

## Quick path (script)

For a fast structural pass, run:

```bash
uv run python scripts/link_pass.py
```

This scans `wiki/concepts/`, `wiki/connections/`, `wiki/qa/`, recent
`daily/` logs, and `wiki/index.md` to build the set of **real** article
titles/slugs. It proposes:

- **Add-link** — verified `[[wikilinks]]` for concepts mentioned in prose
  but not yet linked
- **New-MOC** — cluster maps under `wiki/mocs/<cluster>.md` (from
  `wiki/mocs/_TEMPLATE.md`)
- **Backlink-fix** — missing reciprocal links in `## Related` sections

Unverified targets are logged and skipped — never proposed.

Review the draft, then apply approved items manually or ask the agent to
apply them.

## Agent path (deeper pass)

For richer semantic linking beyond keyword/tag matching:

1. Read `wiki/index.md` and `wiki/log.md`
2. Read all articles in `wiki/concepts/`, `wiki/connections/`, `wiki/qa/`
3. Read recent `daily/` logs (last 7 days)
4. Build the verified title index from files + index — **every proposed
   link must resolve to a real article**
5. Identify:
   - Mentions of existing concepts without `[[wikilinks]]`
   - Thematic clusters (shared tags, co-occurrence, link density)
   - Missing backlinks in `## Related` sections
6. Merge script output with your analysis into `wiki/link-pass-{date}.md`

Use sub-agents for large wikis (30+ articles) — pass file paths only,
not full article content. See `/consolidate` for the sub-agent pattern.

## Draft format

Output to `wiki/link-pass-{YYYY-MM-DD}.md`:

```markdown
# Link Pass Draft — YYYY-MM-DD

> Generated by `/link`. **Not yet applied.** To apply: tell me which
> proposals to keep. To skip everything: delete this file.

## Summary
- N add-link proposals
- N new-MOC proposals
- N backlink-fix proposals

## Add-link proposals

### Proposal 1
**Source (article):** `wiki/concepts/foo.md`
**Add:** `[[Bar]]` → resolves to `bar`
**Context:** ...surrounding sentence...
**Suggested placement:** `## Related` or inline at first mention

## New-MOC proposals

### MOC: Strategy
**Path:** `wiki/mocs/strategy.md`
**Shared tags:** strategy
**Articles:** [[Foo]], [[Bar]], [[Baz]]

## Backlink-fix proposals

### Backlink 1
**Target:** `wiki/concepts/bar.md`
**Add to `## Related`:** `[[foo]]`
```

Order proposals by impact — highest-leverage links first.

## After writing the draft

Tell the founder:

> "I drafted [N] link/MOC proposals at `wiki/link-pass-{date}.md`.
> Gist: [1-line summary]. Review when you have 5 minutes. To apply some
> or all, say which ones — or 'apply all' if it all looks right. To skip
> everything, delete the file."

**Do not apply anything yet.** Wait for the founder to approve.

## Applying approved changes

When the founder approves:

1. Add approved `[[wikilinks]]` to source articles (inline or `## Related`)
2. Create approved MOC files under `wiki/mocs/` using `_TEMPLATE.md`
3. Add approved backlinks to target `## Related` sections
4. Update `wiki/index.md` with new MOC entries
5. Append to `wiki/log.md`:

   ```markdown
   ## YYYY-MM-DD link-pass | Applied N proposals
   - Linked: [[Foo]] → [[Bar]] in `wiki/concepts/foo.md`
   - Created MOC: [[strategy]]
   - Backlink: [[Bar]] ↔ [[Foo]]
   ```

6. Move the draft to `wiki/link-pass-archive/{date}.md` (create dir if needed)

## Article format

Follow the compile agent's article format when adding links:

```markdown
---
title: [Name]
type: concept | entity | connection | exploration
created: [YYYY-MM-DD]
updated: [YYYY-MM-DD]
source: conversation | setup | import | exploration
tags: [comma-separated]
---

# [Name]

[Content with [[wikilinks]] to related articles]

## Related
- [[Related Article 1]]
- [[Related Article 2]]
```

**Wikilink discipline:** Use kebab-case slugs with pipe syntax for display
names when helpful: `[[growth-bottleneck|Growth Bottleneck]]`. Only link
to articles that exist in the verified index.

## Constraints

- **Never auto-apply.** Always require approval.
- **Never hallucinate links.** Verify every target against real files +
  `wiki/index.md` before proposing.
- **Don't edit `daily/` or `imports/`.** Daily logs are inputs only.
- **Don't touch existing MOC content without approval.** Propose via draft.
- **One pass per session.** Link review is mentally light but still needs
  founder eyes.

## When this goes wrong

If you find yourself proposing links to articles that don't exist, stop and
rebuild the verified index. If the draft has 30+ proposals, cut to the top
10 highest-impact and save the rest for the next pass.
